# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for vulnerabilities, credentials, private
data exposure, or other sensitive reports.

Use GitHub's private vulnerability reporting flow:

1. Open the repository's **Security** tab.
2. Select **Report a vulnerability**.
3. Include reproduction steps, affected files or routes, and the expected
   impact.

If private vulnerability reporting is unavailable, contact the maintainer
through the GitHub profile linked from the repository owner account.

## Scope

Security reports are especially helpful for authentication, authorization,
multi-tenant data isolation, dependency vulnerabilities, and accidental secret
or private-data exposure.

## Supported Versions

The latest code on the default branch is the supported development version.
